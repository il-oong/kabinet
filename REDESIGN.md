# Kabinet v1.0 재설계 노트 — 맞춤가구 실무 관점 진단과 개선

## 1. 기존 설계의 문제점 진단

### 치명 (제작 사고로 직결)

**P1. 3D 모델과 커트리스트가 서로 다른 공식을 사용**
가장 심각한 구조적 결함. 도어/서랍 전판 치수를 3D 지오메트리와
커트리스트가 각자 따로 계산했다.

| 부재 | 3D 모델 (기존) | 커트리스트 (기존) | 차이 |
|---|---|---|---|
| 오버레이 도어 폭 (W=1200 기준) | 1200 (갭 0, 전폭) | 내폭+2×9−4 = 1178 | **22mm** |
| 서랍 전판 폭 (W=600) | 596 (풀 오버레이) | 내폭−4 = 560 | **36mm** |
| 서랍 전판 높이 (H=600, 3단) | 196.7 (외곽 기준) | 184.7 (개구 기준) | **12mm** |

도면 보고 조립하면 커트리스트로 재단한 판재가 맞지 않는다.
공장 발주 시 실제 하자로 이어지는 수준.

**P2. 커트리스트 누락/오류 부재가 다수**
- 런 모드(주방 라인 등) 미지원 — EP 높이를 "모듈 높이 합"으로 계산해
  옆으로 나열된 섹션의 높이가 전부 합산됨 (예: 720+720 = 1440mm EP)
- 걸레받이, 세로 분할판, 셀 선반, 셀 서랍, 책상 모듈 부재 전체 누락
- 접이식 도어(4장)를 여닫이(1~2장)로, 미닫이 겹침폭 공식 누락
- 적층 시 하판 공유(suppress_bottom)를 반영하지 않아 자재 과다 산출
- 최상단 상판 폭이 EP 포함 전체 폭 (3D는 EP 사이 폭) — 36mm 오차

**P3. 결방향(무늬결) 정보 파괴**
`mkrow`가 "가로 ≥ 세로"를 보장한다며 치수를 자동 스왑하면서
결방향 컬럼은 그대로 둠 → 세로결 부재가 뒤집혀도 '세로결'로 표기.
LPM/무늬목 재단에서 결방향 오류는 부재 폐기 사유다. **자동 스왑 제거.**

**P4. 언더마운트 슬라이드 클리어런스 오류**
편측 13mm로 설정돼 있었음 — 이는 사이드마운트(12.7mm 볼레일) 값이다.
언더마운트(블럼 탠덤 계열)는 편측 5mm. 기존 값대로 제작하면
서랍통이 좌우 16mm 헐거워 언더마운트 레일 체결 불가.

**P5. 서랍통 깊이가 임의 계산값**
`깊이 − 뒷판 − 후퇴 − 30mm` 같은 임의값으로는 하드웨어 발주가 불가능.
레일은 250/300/350/…/600mm 규격품이므로 서랍통 깊이는
**규격 길이에 스냅**되어야 한다.

### 중대 (품질/시공 하자)

**P6. `has_back` 옵션이 3D에 반영 안 됨** — 스키마에는 있지만
ShelfModule/DrawerModule이 Carcase에 전달하지 않아 "뒷판 없음"
선택이 무시됨. (수정 완료)

**P7. 오버레이 도어 기본 갭 0mm** — 도어가 몸통 전폭을 덮어
인접 모듈/벽과 간섭. 실무 기본은 편측 2mm. (기본값 2mm로 변경,
0 입력 시 플러시 유지 가능)

**P8. EP(측면 마감판)가 카케이스 깊이에서 끝남** — 도어(+20mm 돌출)
옆면이 그대로 노출. 실무에서 EP는 도어 전면까지 나온다.
(`ep.cover_fronts` 기본 true로 도어 전면까지 자동 연장)

**P9. 가동선반이 내폭과 동일 치수** — 여유 0이면 선반이 안 들어간다.
끼임 방지 총 2mm(편측 1mm) 여유 적용. (3D·커트리스트 동일 적용)

**P10. 힌지/32mm 시스템 상수가 정의만 되고 미사용** — 하드웨어
수량 산출에 활용하도록 개선 (힌지 수량표 실사용).

### 구조 (유지보수성)

- 검증이 사실상 없음: 도어 폭 900mm 여닫이, 18T 선반 1100mm 스팬,
  원장(2440×1220) 초과 부재도 조용히 생성됨
- version.rb(0.7.0)와 실제 릴리스(v0.9)가 불일치
- 실패를 조용히 삼키는 `rescue StandardError; nil` (도어 생성 실패가
  흔적 없이 사라짐)

## 2. 재설계 원칙 (30년 실무 관점)

> **"도면 따로, 재단 목록 따로 계산하는 공장은 반드시 사고 난다.
> 치수는 한 곳에서만 계산한다."**

1. **단일 치수 산출 엔진** — `core/fitting.rb` 신설.
   도어/전판/서랍통/선반 치수 공식은 모두 여기에만 존재하고,
   3D 지오메트리와 커트리스트가 같은 함수를 호출한다.
   (레이아웃 함수는 순수 사칙연산이라 SU Length·mm Float 겸용)
2. **하드웨어는 규격품** — 서랍통 깊이는 슬라이드 규격표
   (250~600, 50mm 단위)에 스냅. 규격 미달이면 경고.
3. **재단 목록은 공장 언어로** — 결방향 보존, 엣지밴딩 컬럼,
   하드웨어 수량표(힌지·레일·선반핀·레벨러·손잡이) 동봉.
4. **설계 단계에서 하자 예방** — `core/validation.rb`가
   도어 폭/높이 한계, 선반 처짐 스팬, 원장 초과, 레일 규격 미달을
   생성 시점에 경고.

## 4. 도면 출력 재설계 (v1.0) — "항상 안 되던" 원인과 해법

### 기존 파이프라인이 실패하던 이유 (SketchUp 씬 → PNG → PDF)

**D1. 씬 카메라 저장 순서 버그 (치명)** — `setup_camera`로 카메라를
맞춘 **후에** `model.pages.selected_page = page`를 호출했다.
페이지 선택은 뷰를 그 페이지에 *저장된* (엉뚱한) 카메라로 되돌리므로,
이어지는 `page.update`는 되돌아간 카메라를 저장한다.
→ 정면/측면이 아닌 기본 3D 시점이 씬에 박힘. **순서를
페이지 선택 → 카메라 설정 → update로 교정.**

**D2. 씬 전환 애니메이션 중 캡처** — `selected_page=`는 기본 1초
전환 애니메이션을 유발하는데 `write_image`가 즉시 실행되어 중간
프레임/빈 화면이 찍혔다. → `transition_time = 0` + 페이지 카메라를
`active_view.camera=`로 직접 확정.

**D3. 치수선 전 뷰 중첩** — 치수 태그가 1개라 정면도에 측면 치수가
겹쳐 보이고 `zoom_extents` 범위도 왜곡됐다. → 뷰별 태그
(`Kabinet_Dimensions_front` 등)로 분리, 씬마다 자기 치수만 표시.

**D4. 단면 평면 무한 누적** — deactivate만 하고 지우지 않아 출력할
때마다 모델에 쌓였다. → Kabinet 소유 표시 후 다음 출력 시 제거.

**D5. 실패 무음 처리** — `write_image`가 `rescue → nil`로 실패를
삼켜 원인 파악이 불가능했다. → 콘솔 로깅 추가.

### 새 1차 출력: 발주도면 DXF (모델 무관, 순수 계산)

화면 캡처가 아니라 **spec에서 직접 CAD 도면을 그린다.**
SketchUp 모델 선택도 필요 없다 — 폼에 입력된 값이 곧 도면이다.

- `output/drawing2d.rb` — 정면/측면/평면 2D 투영 (Fitting 공식 공유
  → 3D·커트리스트·도면 삼자 일치)
- `output/dxf.rb` — DXF R12 라이터 (AutoCAD/LibreCAD/CADian 호환,
  한글 CP949 + `$DWGCODEPAGE ANSI_949`)
- `output/order_sheet.rb` — 발주도면 시트: 3각법 배치(평면 위·정면
  아래·측면 우측), 1:1 mm 작도, 도면 틀 + 표제란(품명/규격/재질/날짜)
  + NOTE(단위·엣지·경고), 치수는 건축 관례 사선 틱

도면 내용 (발주 실무 기준 — 과한 디테일 배제):
- 정면도: 외곽 + 도어/전판(개폐 방향 V 표시) + 내부 구조 은선 +
  전체/모듈별 분할 치수
- 측면도: 깊이 단면 윤곽 + 걸레받이 + 뒷판 은선 + 도어 돌출 표시 +
  깊이/총깊이(도어 포함) 치수
- 평면도: 폭×깊이 + EP + 도어 라인 + 섹션 경계
- 레이어 분리: OUTLINE/FRONTS/HIDDEN/SYMBOL/DIM/TEXT/TITLE
  (공장에서 레이어 끄고 켜며 활용)

검증: ezdxf 파서로 감사(audit) 오류 0건, 렌더링 이미지 육안 확인 완료.

## 5. UI 재설계 (v1.0) — 4탭 → 단일 페이지

### 기존 불편 요인
- 4개 탭에 기능 분산: 치수는 어셈블리 탭, 모듈은 모듈 탭, 출력은
  도면 탭 — 값 하나 바꾸고 결과 뽑으려면 탭 3번 이동
- 생성/재생성 버튼이 탭마다 중복, 뭘 눌러야 할지 불명확
- 도면 출력은 EntityID 수동 로드가 선행 조건 (실패의 주 원인)
- 프리셋이 드롭다운 + 별도 탭 27개 버튼으로 이중화
- 입력 결과를 3D 생성 전까지 확인 불가 (깜깜이 입력)

### 새 구조
1. **상단 1줄**: 통합 프리셋 드롭다운 (기본 유형 + 내 프리셋 통합)
2. **실시간 정면도 미리보기** (canvas) — 치수/모듈/도어를 바꾸는
   즉시 반영. 생성 전에 눈으로 확인
3. **기본 정보 카드**: 폭/깊이/총높이/받침 2열 그리드
4. **모듈 구성 카드** (기존 카드 UI 유지)
5. 상판·EP·걸레받이·런 모드는 **접이식 '구조 옵션'**으로 정리
   (기본값이 실무 표준이라 평소엔 열 필요 없음)
6. **하단 고정 액션바**: [선택 불러오기] [모델 반영(생성/재생성
   자동 판단)] [발주도면 DXF] [커트리스트] — 어디서든 한 번에 출력
7. 씬 도면(PNG/PDF)은 보조 출력으로 접이식 카드에 유지

## 6. 전 구성 전수 감사 (v1.0.1) — 프리셋 27종 파이프라인 검증

27개 기본 프리셋(상부장 포함)을 정규화→검증→커트리스트→하드웨어→
DXF 도면 전 파이프라인에 통과시키고 도면을 육안 검수했다.

### 발견·수정한 문제

**B1. 침대장: 걸레받이·상판이 침대 공간(bed_gap)을 관통 (치명)**
런 모드에서 걸레받이와 상판을 전폭 통짜 1장으로 생성 — 침대 놓을
자리 바닥에 판이 지나가는 구조였다. 3D·커트리스트·도면·UI 미리보기
모두 동일 버그. → bed_gap 제외 연속 구간별 분할 생성으로 수정
(예: 침대장 걸레받이 600×80 두 장). 브릿지가 필요한 경우를 위한
안내 경고 추가.

**B2. TV장 프리셋: 실무 불가 구성**
- 1764mm 통짜 서랍 3단 — 레일 하중/전판 휨으로 제작 불가
  → 세로 분할 3칸 × 서랍 1단 (칸당 564mm)
- 상부 여닫이 878mm/짝 — 힌지 한계 초과 → 중앙 분할 오픈 선반 2칸

**B3. 선반 처짐 구성 정리 (18T 무지지 스팬 초과)**
- 수납장1200(1164) → 중앙 분할판 + 셀 선반 3+3
- 자녀방 상부장 3종(1170) → 중앙 분할 + 셀 선반
- 자녀방 붙박이장·거실 런(1164) → 25T 상향
- 책장(864, 책 하중) → 25T 상향
- 슬라이딩장(1564) → 선반 제거, 행거 중심 구성

**B4. 슬라이딩장 행거봉 높이 오류** — 봉이 500/1000mm(옷이 바닥에
끌리는 높이)였음 → 2단 행거 표준 1850/930mm로 수정.

**B5. 검증 정확도** — 스택 모드 경고가 카케이스 보정 전 폭으로
계산되던 것을 실제 시공 폭 기준으로 수정. 신규 경고 추가:
서랍 개구폭 1000mm 초과, 행거봉 스팬 1200mm 초과, 전폭 선반×세로
분할판 동시 사용(관통 충돌), bed_gap 위 상판 분할 안내.

**B6. 도면·미리보기 정합** — 셀 선반 높이 기준이 도면(+18mm)과 3D가
달랐던 것 통일. 프리셋 로드 시 모듈 폭을 카케이스 내부 폭으로 자동
동기화(카드 표시값 = 실제 제작값).

감사 결과: 27/27 파이프라인 통과, DXF 27종 파서 감사 오류 0건.
잔여 경고는 전부 정보성(주방장 830~870mm 스팬 등 실무 통용 범위).

## 7. 남은 과제 (다음 단계 제안)

- **32mm 시스템 홀 지오메트리** — 상수는 준비됨. 측판에 선반핀
  홀 마킹(또는 도면 표기)을 추가하면 CNC 연동 가능
- **적층 vs 통측판 구조 선택** — 현재는 "박스 적층 + 하판 공유"
  하이브리드. 붙박이장은 통측판+고정선반 방식 옵션이 필요
- **뒷판 시공법 옵션** — 현재 끼움식(전면 10mm 후퇴)만 표현.
  홈삽입(다도)/후부착 선택과 그에 따른 재단 여유 반영
- **네스팅(재단 배치) 최적화** — 원장 대비 수율 계산
- **UI 경고 패널** — 현재는 생성 완료 메시지에 텍스트로 표시
