# Kabinet v1.0 재설계 노트 — 맞춤가구 실무 관점 진단과 개선

## 1. 기존 설계의 문제점 진단

### 치명 (제작 사고로 직결)

**P1. 3D 모델과 커트리스트가 서로 다른 공식을 사용**
가장 심각한 구조적 결함. 도어/서랍 전판 치수를 3D 지오메트리와
커트리스트가 각자 따로 계산했다.

| 부재 | 3D 모델 (기존) | 커트리스트 (기존) | 차이 |
|---|---|---|---|
| 오버레이 도어 폭 (W=1200 기준) | 1200 (갭 0, 전폭) | 내폭+2×9−4 = 1178 | **22mm** |
| 서랍 전판 폭 (W=600) | 596 (풀 오버레이) | 내폭−4 = 560 | **36mm** |
| 서랍 전판 높이 (H=600, 3단) | 196.7 (외곽 기준) | 184.7 (개구 기준) | **12mm** |

도면 보고 조립하면 커트리스트로 재단한 판재가 맞지 않는다.
공장 발주 시 실제 하자로 이어지는 수준.

**P2. 커트리스트 누락/오류 부재가 다수**
- 런 모드(주방 라인 등) 미지원 — EP 높이를 "모듈 높이 합"으로 계산해
  옆으로 나열된 섹션의 높이가 전부 합산됨 (예: 720+720 = 1440mm EP)
- 걸레받이, 세로 분할판, 셀 선반, 셀 서랍, 책상 모듈 부재 전체 누락
- 접이식 도어(4장)를 여닫이(1~2장)로, 미닫이 겹침폭 공식 누락
- 적층 시 하판 공유(suppress_bottom)를 반영하지 않아 자재 과다 산출
- 최상단 상판 폭이 EP 포함 전체 폭 (3D는 EP 사이 폭) — 36mm 오차

**P3. 결방향(무늬결) 정보 파괴**
`mkrow`가 "가로 ≥ 세로"를 보장한다며 치수를 자동 스왑하면서
결방향 컬럼은 그대로 둠 → 세로결 부재가 뒤집혀도 '세로결'로 표기.
LPM/무늬목 재단에서 결방향 오류는 부재 폐기 사유다. **자동 스왑 제거.**

**P4. 언더마운트 슬라이드 클리어런스 오류**
편측 13mm로 설정돼 있었음 — 이는 사이드마운트(12.7mm 볼레일) 값이다.
언더마운트(블럼 탠덤 계열)는 편측 5mm. 기존 값대로 제작하면
서랍통이 좌우 16mm 헐거워 언더마운트 레일 체결 불가.

**P5. 서랍통 깊이가 임의 계산값**
`깊이 − 뒷판 − 후퇴 − 30mm` 같은 임의값으로는 하드웨어 발주가 불가능.
레일은 250/300/350/…/600mm 규격품이므로 서랍통 깊이는
**규격 길이에 스냅**되어야 한다.

### 중대 (품질/시공 하자)

**P6. `has_back` 옵션이 3D에 반영 안 됨** — 스키마에는 있지만
ShelfModule/DrawerModule이 Carcase에 전달하지 않아 "뒷판 없음"
선택이 무시됨. (수정 완료)

**P7. 오버레이 도어 기본 갭 0mm** — 도어가 몸통 전폭을 덮어
인접 모듈/벽과 간섭. 실무 기본은 편측 2mm. (기본값 2mm로 변경,
0 입력 시 플러시 유지 가능)

**P8. EP(측면 마감판)가 카케이스 깊이에서 끝남** — 도어(+20mm 돌출)
옆면이 그대로 노출. 실무에서 EP는 도어 전면까지 나온다.
(`ep.cover_fronts` 기본 true로 도어 전면까지 자동 연장)

**P9. 가동선반이 내폭과 동일 치수** — 여유 0이면 선반이 안 들어간다.
끼임 방지 총 2mm(편측 1mm) 여유 적용. (3D·커트리스트 동일 적용)

**P10. 힌지/32mm 시스템 상수가 정의만 되고 미사용** — 하드웨어
수량 산출에 활용하도록 개선 (힌지 수량표 실사용).

### 구조 (유지보수성)

- 검증이 사실상 없음: 도어 폭 900mm 여닫이, 18T 선반 1100mm 스팬,
  원장(2440×1220) 초과 부재도 조용히 생성됨
- version.rb(0.7.0)와 실제 릴리스(v0.9)가 불일치
- 실패를 조용히 삼키는 `rescue StandardError; nil` (도어 생성 실패가
  흔적 없이 사라짐)

## 2. 재설계 원칙 (30년 실무 관점)

> **"도면 따로, 재단 목록 따로 계산하는 공장은 반드시 사고 난다.
> 치수는 한 곳에서만 계산한다."**

1. **단일 치수 산출 엔진** — `core/fitting.rb` 신설.
   도어/전판/서랍통/선반 치수 공식은 모두 여기에만 존재하고,
   3D 지오메트리와 커트리스트가 같은 함수를 호출한다.
   (레이아웃 함수는 순수 사칙연산이라 SU Length·mm Float 겸용)
2. **하드웨어는 규격품** — 서랍통 깊이는 슬라이드 규격표
   (250~600, 50mm 단위)에 스냅. 규격 미달이면 경고.
3. **재단 목록은 공장 언어로** — 결방향 보존, 엣지밴딩 컬럼,
   하드웨어 수량표(힌지·레일·선반핀·레벨러·손잡이) 동봉.
4. **설계 단계에서 하자 예방** — `core/validation.rb`가
   도어 폭/높이 한계, 선반 처짐 스팬, 원장 초과, 레일 규격 미달을
   생성 시점에 경고.

## 4. 남은 과제 (다음 단계 제안)

- **32mm 시스템 홀 지오메트리** — 상수는 준비됨. 측판에 선반핀
  홀 마킹(또는 도면 표기)을 추가하면 CNC 연동 가능
- **적층 vs 통측판 구조 선택** — 현재는 "박스 적층 + 하판 공유"
  하이브리드. 붙박이장은 통측판+고정선반 방식 옵션이 필요
- **뒷판 시공법 옵션** — 현재 끼움식(전면 10mm 후퇴)만 표현.
  홈삽입(다도)/후부착 선택과 그에 따른 재단 여유 반영
- **네스팅(재단 배치) 최적화** — 원장 대비 수율 계산
- **UI 경고 패널** — 현재는 생성 완료 메시지에 텍스트로 표시
